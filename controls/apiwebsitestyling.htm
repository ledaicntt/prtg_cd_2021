Styling the Web Interface using CSS, JS, and HTML Injection

PRTG users can use their own logos and/or corporate colors inside the AJAX Web GUI by adding their own HTML and CSS code to PRTG's web interface. For the AJAX Web GUI it is also possible to add your own JavaScript code, too.

Limited Support

This feature should only be used by experienced web designers with a good knowledge of HTML, JavaScript, and CSS. Due to the complexity of PRTG's code the Paessler support team cannot support you regarding the actual HTML, JavaScript, and/or CSS code that you need to use.

Note: New versions of PRTG may render the web pages with a different HTML structure and may require changes for your CSS statements (however, we will try to avoid such situations).

How does it work?

When web pages are created, PRTG uses various include files from the webroot sub folder of your PRTG program directory. You can add your own HTML, CSS, and JavaScript by editing the following files for the AJAX Web GUI.

Interface Folder and File Contents Will be inserted
AJAX Web GUI webroot\includes\htmlheader_custom.htm HTML at the top of the page, just after the <BODY> tag
AJAX Web GUI webroot\includes\htmlfooter_custom.htm HTML at the bottom of the page, just before the </BODY> tag
AJAX Web GUI webroot\css\styles_custom.css CSS at the end of PRTG's CSS file for the AJAX Web GUI
AJAX Web GUI webroot\javascript\scripts_custom.js JavaScript at the end of PRTG's JavaScript file for the AJAX Web GUI

The contents of these files are then added at the respective positions. By creating these files and adding your own HTML and CSS statements (e.g. by using a text editor) you can append your own HTML and overwrite the built-in CSS statements to change PRTG's look and feel. Custom JavaScript gives you even more possibilities.

After adding/changing the file(s) a simple "full reload" of your browser window (Shift-F5) will instantly use the new code and you should be able to see the changes immediately.

Note: Please only modify the files mentioned above. Changing PRTG's built-in HTML and CSS files is not recommended and not supported. All changes to PRTG's other files will be overwritten with new files as soon as a new PRTG version is installed.

Writing CSS Code

For some common redesign tasks (colors, logos, margins) we have included sample statements in the CSS code below. If you have additional design requirements we recommend that you use a tool like FireBug, Chrome Developer Tools or any other web page analyzer tool to dig into PRTG's existing HTML code structure and CSS statements in order to find out what CSS statements you must add/change. 

/* Sample CSS redesign file for PRTG Network Monitor, works with V13.2.3.x or later */
    /* Created May 2013 by Dirk Paessler */
    /* This styling is not meant to look pretty, 
        it serves as sample code on how to change the style */

    /* How to use:
    - Create file "styles_custom.css" in the \prtg network monitor\webroot\css folder
    - insert the code below and edit as desired
    - reload your browser */

body  /*change base font and size */
{
  font-family:"Times New Roman",times,serif;
  font-size: 10px;
}

#main /* Change background color */
{
  background:#666;
}

#header_menu /*change top header background color*/
{
  background:#eee;
}

#header_breadcrumbs /*change top header background color*/
{
background:#000;
color: #333;
}
