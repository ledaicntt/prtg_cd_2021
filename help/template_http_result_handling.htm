Define what the sensor will do with the data loaded at the given URL. Choose between:

  • Discard HTML result: Do not store the requested data.
  • Store latest HTML result: Store the last result of the requested data to the Logs (Sensors) directory in the PRTG data folder on the probe system the sensor is running on (on the Master node if in a cluster). File name: Result of Sensor [ID].txt. This is for debugging purposes, especially in combination with content checks. The file will be overridden with each scanning interval.
    For information on how to find the folder used for storage, see section Data Storage.

 

This option is not available on cloud probes in PRTG in the cloud.
